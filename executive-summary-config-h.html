What Was Done

Four directives, one deliverable

  • Install and authenticate the Snyk CLI.
  • Run the SAST scan via snyk code test.
  • Run the dependency scan via snyk test, with SBOM fallback.
  • Normalize the two artifacts into findings-config-h.json.

Scope of the scan

  • src/: 88 Rust files across nine subsystems.
  • build.rs: the build script.
  • Cargo.toml plus Cargo.lock: the dependency manifest.
  • tests/, benches/, examples/: per Snyk Code defaults.

Why This Approach

Rust meets Snyk: two deviations

  • SBOM fallback: cargo cyclonedx plus snyk sbom test. Cargo manifests are not natively supported.
  • SAST gating: Rust on Snyk Code is Early Access. Without entitlement, the SAST contribution is [].
  • Both deviations are deterministic and documented in decisions-config-h.md.

What Changed Architecturally

Zero source-tree modifications

  • src/, tests/, benches/, examples/ untouched.
  • Cargo.toml, Cargo.lock, build.rs untouched.
  • Six new artifacts written at the workspace root.
  • Host state: Snyk CLI installed globally only.

What Risks Exist

The deliverable lists findings. Reading it requires understanding the scan's authentication state and the truncation rules.

Constraints to know

  • Snyk requires network access; no offline mode.
  • SNYK_TOKEN must be present in the runtime environment.
  • Exit codes and wall-clock durations are recorded in decisions-config-h.md.
  • Per-config result; no cross-tool reconciliation in scope.
