## Deviations from Literal Requirements

The workflow diverged from the literal user prompt in the following places. Each entry uses the format `Deviation · Justification`.

- **Three sibling artifacts rather than one** · The user statement `~0 files modified | 1 new file` names `findings-config-i.json` only. The Explainability rule (AAP §0.7.1) and the Executive Presentation rule (AAP §0.7.2) apply globally to every deliverable on the Blitzy platform and add two further required files (`decision-log-config-i.md`, `executive-summary-config-i.html`), per the reconciliation note in AAP §0.7.4.
- **Scanner authentication via a short-lived token instead of `sonar.login=admin` / `sonar.password=admin`** · SonarQube Community Build `26.5.0.122743` deprecated username-password authentication in the scanner protocol (captured verbatim in `blitzy/qa-evidence/checkpoint1-evidence.txt`: `Not authorized. Please check the user token in property 'sonar.token' or 'sonar.login' (deprecated).`). The workflow uses the user-supplied `admin/admin` credentials as the only authentication input, mints an ephemeral token from those credentials via `POST /api/user_tokens/generate`, and destroys both the token and the container at teardown.
- **No rule-description fallback was exercised and no `CWE-UNKNOWN` sentinel was emitted** · The raw issue count was `0`, so neither the tag scan nor the `/api/rules/show` fallback ran. The count of `CWE-UNKNOWN` records is therefore `0`. The cascade code path is exercised end-to-end by adversarial fixtures in `blitzy/qa-evidence/checkpoint1-evidence.txt` (3 rule lookups across 4 issues, 2 memoized hits), confirming the cascade behaves correctly on synthetic data with all three branches reachable.
- **`cargo` exposed on `PATH` before scanner invocation** · Directive D3 does not mention `cargo` PATH management, but the Rust analyzer's full ruleset requires `cargo` on `PATH` per AAP §0.8.1. Sourcing `$HOME/.cargo/env` at the top of `run_sonarqube_scan.sh` leaves the literal scanner invocation byte-identical while ensuring the Clippy sensor (the bulk of the 85-rule Rust ruleset) actually runs.
